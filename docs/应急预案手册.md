# 应急预案手册

## 快速判断

```
大屏字幕正常？
  ├─ 是 → L1 正常，继续监控
  └─ 否 → 字幕停止更新？
           ├─ 状态显示"重连中"→ L3 网络/ASR断联
           └─ 字幕有中文但无英文 → L2 翻译异常
```

---

## L1 — 正常运行 🟢

**触发条件**：各项指标平稳，延迟 ≤3s，双语字幕正常输出

**操作**：每30分钟巡检一次，无需干预

---

## L2 — 翻译降级 🟡

**触发条件**：
- 英文显示 `[译文生成中...]` 持续出现
- 网关日志出现 `Translation timeout` 或 `Qwen HTTP 429`

**自动处理**：系统自动切换"原文直出 + 占位"，中文字幕不中断

**手动恢复**：
```bash
# 检查 API 余额
# 登录 https://dashscope.console.aliyun.com/ → 用量统计

# 临时切换更快的模型
# 编辑 gateway/.env → TRANSLATE_MODEL=qwen-turbo
# 然后重启网关（<30秒）
```

**向观众说明**（如需）："字幕系统切换至中文直出模式，英文翻译稍后恢复。"

---

## L3 — 网络/ASR 断联 🟡

**触发条件**：
- 大屏状态显示 `● 重连中...`（黄色）
- 网关日志出现 `ASR WebSocket closed` 或 `ASR/audio error`

**自动处理**：网关 ASR 循环每3秒自动尝试重连，大屏 WebSocket 每2秒重连

**手动操作（如自动恢复失败）**：
```bash
# 步骤1：切换备用网络（5G热点）
# 步骤2：重启网关
Ctrl+C
python gateway/main.py

# 步骤3：刷新大屏浏览器
F5
```

预计恢复时间：**<30秒**

---

## 熔断 — 强制介入 🔴

**触发条件**：
- 连续3分钟错误率 >15%
- CPU >90% 持续
- 网关进程崩溃

**操作**：
1. `Ctrl+C` 停止网关
2. 打开备用静态字幕页（预先准备的核心术语对照表）：
   ```
   screen/fallback_static.html  （会前准备好，展示核心议题双语术语）
   ```
3. 联系技术负责人

**人工兜底**：速记员将内容实时输入共享文档（腾讯文档/飞书），投屏显示

```bash
# 查看实时日志

# 测试 API 连通性（不启动音频）
set DISABLE_AUDIO=1 && python gateway/main.py

notepad gateway/terms.json
```
